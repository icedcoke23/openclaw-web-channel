# OpenClaw Web Channel 开发路线图

## 项目概述

OpenClaw Web Channel 是一个生产级的 OpenClaw AI 助手网关 Web 仪表盘，提供实时聊天、状态监控、会话管理、配置编辑、日志查看、技能管理和节点管理等功能。

**当前版本**: v3.0.0  
**技术栈**: React 19 + TypeScript + Vite 6 + Tailwind CSS 4 + Express + WebSocket

---

## 开发原则

1. **质量优先**: 每个功能必须包含完整实现、错误处理、TypeScript 类型
2. **用户体验**: 响应式设计、流畅动画、清晰反馈
3. **代码规范**: 组件化、模块化、可测试
4. **向后兼容**: 不破坏现有功能
5. **中文优先**: 所有 UI 文本使用中文

## 阶段划分

### Phase 1: P0 核心功能缺失（预计 2-3 天）

优先级最高，直接影响核心使用体验。

#### 1.1 图片预览功能
- **目标**: 聊天附件支持内联图片预览和灯箱查看
- **实现**:
  - 修改 `ChatPanel.tsx` 的附件展示逻辑
  - 图片类型附件显示缩略图
  - 点击缩略图打开灯箱（Lightbox）全屏查看
  - 支持图片缩放、拖拽、下载
- **验收标准**:
  - PNG/JPG/GIF/WebP 图片正确显示缩略图
  - 灯箱支持 ESC 关闭、点击遮罩关闭
  - 响应式布局，移动端适配

#### 1.2 定时任务完整 CRUD
- **目标**: 支持创建和删除定时任务
- **实现**:
  - 后端: 添加 `POST /api/cron` 创建端点
  - 后端: 添加 `DELETE /api/cron/:id` 删除端点
  - 前端: `CronPanel.tsx` 新增创建表单（名称、Cron表达式、启用状态）
  - 前端: 每行添加删除按钮（带确认弹窗）
- **验收标准**:
  - 成功创建新定时任务
  - 成功删除定时任务
  - Cron 表达式实时验证
  - 操作后有 Toast 反馈

#### 1.3 磁盘使用率监控
- **目标**: Dashboard 显示磁盘使用率
- **实现**:
  - 后端: 修改 `/api/status` 或新增 `/api/system/resources` 返回磁盘信息
  - 前端: DashboardPanel 添加 DISK 进度条卡片
  - 显示总容量、已用、可用、使用率百分比
- **验收标准**:
  - 正确显示磁盘使用率
  - 使用率超过 80% 显示警告色（橙色）
  - 使用率超过 90% 显示危险色（红色）

#### 1.4 会话搜索功能
- **目标**: SessionsPanel 支持按标题/ID搜索过滤
- **实现**:
  - 添加搜索输入框
  - 实时过滤会话列表
  - 支持模糊匹配
  - 高亮匹配文本
- **验收标准**:
  - 输入关键词实时过滤
  - 空结果显示"未找到会话"提示
  - 搜索性能流畅（<100ms）

### Phase 2: P1 专业功能增强（预计 3-4 天）

提升专业用户的使用体验。

#### 2.1 费用追踪系统
- **目标**: 显示每日花费、预算管理
- **实现**:
  - 后端: 新增 `/api/metrics/cost` 端点，返回费用数据
  - 前端: Dashboard 添加费用卡片（今日花费、本月花费）
  - 添加预算设置和进度条
  - 接近预算上限时显示警告
- **验收标准**:
  - 正确显示费用数据
  - 预算进度条颜色随使用率变化
  - 支持设置月度预算

#### 2.2 Token 使用图表
- **目标**: 可视化展示 Token 使用情况
- **实现**:
  - 引入图表库（Recharts 或 Chart.js）
  - 24小时 Token 使用趋势图（Sparkline）
  - 按模型分布的饼图
  - 输入/输出 Token 对比柱状图
- **验收标准**:
  - 图表响应式，支持暗色主题
  - 数据实时更新
  - 悬停显示详细数值

#### 2.3 配置 Diff 预览
- **目标**: 保存配置前显示变更对比
- **实现**:
  - 修改 `ConfigPanel.tsx`
  - 保存前弹出 Diff 预览模态框
  - 使用 diff 库（如 `diff` 或 `fast-diff`）生成差异
  - 高亮新增/删除/修改的行
- **验收标准**:
  - 清晰显示新旧配置差异
  - 支持确认保存或取消
  - 大配置也能流畅显示

#### 2.4 实时 JSON 校验
- **目标**: ConfigPanel JSON 编辑器支持实时语法检查
- **实现**:
  - 引入代码编辑器组件（如 `@monaco-editor/react` 轻量版，或 `react-simple-code-editor`）
  - 实时验证 JSON 语法
  - 错误行高亮
  - 显示错误提示信息
- **验收标准**:
  - 语法错误实时显示
  - 行号正确标注
  - 性能不影响输入流畅度

#### 2.5 聊天导出功能
- **目标**: 支持导出聊天记录
- **实现**:
  - ChatPanel 添加导出按钮
  - 支持导出为 Markdown、JSON、TXT
  - 导出包含完整对话内容、时间戳、Token 统计
- **验收标准**:
  - 三种格式都能正确导出
  - Markdown 格式美观可读
  - JSON 格式包含完整元数据

#### 2.6 聊天历史搜索
- **目标**: 全局搜索聊天内容
- **实现**:
  - 后端: 新增 `/api/search` 或扩展 `/api/history` 支持搜索参数
  - 前端: 添加全局搜索框（可放在 Header 或独立页面）
  - 支持按关键词、时间范围、会话筛选
  - 搜索结果高亮显示
- **验收标准**:
  - 搜索响应 < 500ms
  - 结果分页显示
  - 点击结果跳转到对应消息

#### 2.7 技能详情弹窗
- **目标**: SkillsPanel 支持查看技能详情
- **实现**:
  - 点击技能卡片打开详情弹窗
  - 显示完整描述、参数说明、使用示例
  - 如果有 README，渲染 Markdown 显示
- **验收标准**:
  - 弹窗信息完整
  - README 渲染正确

#### 2.8 明/暗主题切换
- **目标**: 支持亮色/暗色主题切换
- **实现**:
  - 扩展 CSS 变量系统，定义亮色主题色值
  - 添加主题切换按钮（Header 或设置页）
  - 保存用户偏好到 localStorage
  - 支持跟随系统主题
- **验收标准**:
  - 亮色主题美观可读
  - 切换流畅无闪烁
  - 刷新后保持主题选择

#### 2.9 键盘快捷键增强
- **目标**: 丰富键盘快捷键系统
- **实现**:
  - `g + h` → 跳转到首页
  - `g + c` → 跳转到聊天
  - `g + s` → 跳转到会话
  - `g + d` → 跳转到仪表盘
  - `g + l` → 跳转到日志
  - `Esc` → 关闭弹窗/退出搜索
  - `?` → 显示快捷键帮助
- **验收标准**:
  - 快捷键响应灵敏
  - 不与其他应用冲突
  - 快捷键帮助页面清晰

### Phase 3: P2 增强功能（预计 2-3 天）

提升产品完整性和用户体验。

#### 3.1 PWA 离线支持
- **目标**: 支持安装为桌面应用，基础离线功能
- **实现**:
  - 创建 `manifest.json`
  - 添加 Service Worker（使用 Workbox）
  - 缓存静态资源
  - 离线时显示友好提示
- **验收标准**:
  - 可安装为 PWA
  - 离线能打开界面（功能受限）

#### 3.2 多语言 i18n
- **目标**: 支持中英文切换
- **实现**:
  - 引入 i18n 框架（`react-i18next`）
  - 提取所有中文文本到语言文件
  - 添加语言切换器
  - 默认中文，支持英文
- **验收标准**:
  - 切换语言后所有文本更新
  - 刷新后保持语言选择

#### 3.3 浏览器通知
- **目标**: 新消息时推送浏览器通知
- **实现**:
  - 请求 Notification 权限
  - 新消息时发送通知
  - 点击通知跳转到对应会话
  - 支持关闭通知
- **验收标准**:
  - 正确请求权限
  - 通知内容准确
  - 点击能正确跳转

#### 3.4 语音输入
- **目标**: 支持语音转文字输入
- **实现**:
  - 使用 Web Speech API
  - 添加麦克风按钮
  - 录音时显示动画反馈
  - 支持中文语音识别
- **验收标准**:
  - 在支持的浏览器中正常工作
  - 识别准确率可接受
  - UI 反馈清晰

### Phase 4: 高级功能（预计 4-5 天）

对标社区标杆（LvcidPsyhe）的高级功能。

#### 4.1 设备管理
- **目标**: 管理配对设备
- **实现**:
  - 后端: 新增 `/api/nodes/devices` 等端点
  - 前端: 新增 DevicesPanel 或扩展 NodesPanel
  - 显示配对设备列表
  - 支持审批、撤销、令牌轮换
- **验收标准**:
  - 完整设备生命周期管理

#### 4.2 网关自动发现
- **目标**: 自动发现局域网内的 OpenClaw 网关
- **实现**:
  - 使用 mDNS/Zeroconf（Bonjour）
  - 自动扫描并列出发现的网关
  - 一键连接发现的网关
- **验收标准**:
  - 能发现同一局域网内的网关

#### 4.3 记忆/上下文可视化
- **目标**: 可视化展示 AI 的记忆和上下文
- **实现**:
  - 新增 MemoryPanel
  - 显示当前会话的上下文窗口
  - 可视化记忆结构
- **验收标准**:
  - 清晰展示上下文使用情况

#### 4.4 Canvas 渲染支持
- **目标**: 支持渲染 OpenClaw 的 Canvas 输出
- **实现**:
  - 扩展 ChatPanel 消息渲染
  - 支持 Canvas/WebGL 组件
  - 用户可交互控制
- **验收标准**:
  - 正确渲染 Canvas 内容

#### 4.5 Flight Deck 驾驶舱
- **目标**: 多面板集成的首页
- **实现**:
  - 重新设计 Dashboard 为驾驶舱布局
  - 多个可拖拽/调整大小的面板
  - 实时数据流展示
  - 类似 LvcidPsyhe 的 Flight Deck
- **验收标准**:
  - 信息密度高但不杂乱
  - 关键指标一目了然

#### 4.6 独立功能页面
- **目标**: 新增 Metrics、Debug、Files、Docs 页面
- **实现**:
  - **Metrics 页**: 详细的指标图表和统计
  - **Debug 页**: 网关连接测试、会话检查器
  - **Files 页**: 工作区文件浏览器
  - **Docs 页**: 内置文档和 API 文档
- **验收标准**:
  - 每个页面功能完整
